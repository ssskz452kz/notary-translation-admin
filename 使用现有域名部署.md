# 使用现有域名 qazaq.fyi 部署管理后台

## 方案：使用子域名（推荐）

你可以创建一个**子域名**专门给管理后台用，比如：
- `admin.qazaq.fyi` - 管理后台
- `notary.qazaq.fyi` - 公证翻译后台
- `backend.qazaq.fyi` - 后台管理

这样主域名 `qazaq.fyi` 可以继续用于其他用途。

---

## 部署步骤

### 1. 在 Vercel 创建新项目

1. 登录 Vercel：https://vercel.com
2. 点击 **"Add New Project"**
3. 选择导入方式：
   - **方式 A（推荐）**：如果代码在 GitHub
     - 选择你的 GitHub 仓库
     - 或创建新仓库上传 `notary-translation-admin` 文件夹
   - **方式 B**：直接上传
     - 选择 "Upload" 或拖拽整个 `notary-translation-admin` 文件夹

### 2. 配置项目设置

- **Framework Preset**：选择 "Other" 或 "Static"
- **Root Directory**：`notary-translation-admin`（如果上传的是整个文件夹）
- **Build Command**：留空（纯静态文件，无需构建）
- **Output Directory**：`.`（当前目录）

### 3. 添加子域名

部署完成后：

1. 进入项目设置 → **Settings** → **Domains**
2. 点击 **"Add Domain"**
3. 输入子域名，例如：`admin.qazaq.fyi`
4. Vercel 会显示需要配置的 DNS 记录（通常是 CNAME）

### 4. 配置 DNS（在你的域名服务商）

1. 登录你的域名服务商（GoDaddy、Namecheap、阿里云等）
2. 找到 DNS 管理/域名解析
3. 添加一条 **CNAME 记录**：
   - **主机记录**：`admin`（或你想要的子域名前缀）
   - **记录值**：Vercel 提供的 CNAME 值（通常是 `cname.vercel-dns.com` 或类似）
   - **TTL**：自动或 3600

4. 等待 DNS 生效（通常几分钟到几小时）

### 5. 完成

DNS 生效后，访问 `admin.qazaq.fyi` 就能看到管理后台了！

---

## 或者：使用主域名 + 路径

如果你不想用子域名，也可以：
- 部署到 `qazaq.fyi/admin` 或 `qazaq.fyi/notary-admin`
- 在 Vercel 项目设置中配置 **Rewrites** 规则

但子域名更简洁，推荐用子域名。

---

## 安全建议

部署到公网后，`config.js` 中的 `serviceRoleKey` 会暴露在前端代码中。

**建议措施**：

1. **使用 Vercel 密码保护**（Vercel Pro 功能）
   - 项目设置 → **Settings** → **Password Protection**
   - 设置一个密码，访问网站时需要输入

2. **或使用 Cloudflare Access**（免费版支持 50 用户）
   - 在 Cloudflare 配置访问策略
   - 只有授权邮箱才能访问

3. **定期检查 Supabase 日志**
   - 在 Supabase Dashboard → Logs 查看 API 调用
   - 发现异常及时更换 `serviceRoleKey`

## 快速开始

**最快的方式**：

1. 在 Vercel 创建新项目，上传 `notary-translation-admin` 文件夹
2. 部署完成后，在项目设置 → Domains 添加 `admin.qazaq.fyi`
3. 在你的域名服务商添加 CNAME 记录指向 Vercel
4. 等待 DNS 生效，完成！

**预计时间**：10-15 分钟（主要是等 DNS 生效）
