# 修复 Vercel 404 错误

## 问题
访问 `admin.qazaq.fyi` 或 `notary-translation-admin.vercel.app` 显示 404 错误。

## 原因
Vercel 可能没有正确识别静态文件结构，或者项目配置不正确。

## 解决方案

### 方案 1：添加 vercel.json 配置文件（已创建）

我已经创建了 `vercel.json` 文件，它会：
- 告诉 Vercel 这是静态网站
- 配置路由重写，所有请求都指向 `index.html`
- 设置缓存头

**下一步**：
1. 提交 `vercel.json` 到 GitHub
2. Vercel 会自动重新部署
3. 等待部署完成（通常 1-2 分钟）

### 方案 2：检查 Vercel 项目设置

如果添加 `vercel.json` 后仍有问题，检查项目设置：

1. 进入 Vercel 项目 → **Settings** → **General**
2. 确认以下设置：
   - **Framework Preset**: `Other` 或 `Static`
   - **Root Directory**: `.`（如果文件在根目录）或留空
   - **Build Command**: 留空
   - **Output Directory**: `.` 或留空
   - **Install Command**: 留空

### 方案 3：手动触发重新部署

1. 在 Vercel 项目页面
2. 点击 **"Deployments"** 标签
3. 找到最新的部署
4. 点击右侧的 **"..."** → **"Redeploy"**

---

## 验证步骤

部署完成后：
1. 等待 1-2 分钟让部署完成
2. 访问 `https://admin.qazaq.fyi`
3. 应该能看到登录页面，而不是 404

---

## 如果还是 404

检查：
1. **文件是否都在根目录**：确保 `index.html`、`dashboard.html` 等文件都在仓库根目录
2. **GitHub 仓库结构**：如果文件在子文件夹，需要在 Vercel 设置中指定 **Root Directory**
3. **浏览器缓存**：清除缓存或使用无痕模式访问
